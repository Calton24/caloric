# iOS Build Setup & Troubleshooting

This document explains the iOS CocoaPods configuration required for React Native 0.81.5 with modern Xcode (26+).

## Problem Overview

React Native 0.81.5 ships with third-party dependencies (glog, boost, etc.) that CocoaPods normally resolves from trunk (the centralized spec repository). However:

1. **Trunk specs are outdated** - glog 0.3.4 on trunk tries to build for armv7 (discontinued in 2019)
2. **Modern Xcode incompatibility** - Xcode 26+ removes support for ancient architectures and build tools
3. **React Native provides updated specs** - RN bundles compatible versions in `node_modules/react-native/third-party-podspecs/`

## The Fix: Local Podspec Overrides

The `ios/Podfile` **MUST** include these explicit local podspec references:

```ruby
target 'YourAppName' do
  use_expo_modules!

  # Force local podspecs for RN third-party deps to avoid trunk resolution issues in CI
  # This prevents CocoaPods from pulling ancient versions (e.g., glog 0.3.4) that fail on Xcode 26+
  pod 'RCT-Folly', :podspec => '../node_modules/react-native/third-party-podspecs/RCT-Folly.podspec'
  pod 'boost', :podspec => '../node_modules/react-native/third-party-podspecs/boost.podspec'
  pod 'glog', :podspec => '../node_modules/react-native/third-party-podspecs/glog.podspec'
  pod 'fast_float', :podspec => '../node_modules/react-native/third-party-podspecs/fast_float.podspec'
  pod 'fmt', :podspec => '../node_modules/react-native/third-party-podspecs/fmt.podspec'

  # ...rest of configuration
end
```

### Why This Works

- **Bypasses trunk** - CocoaPods uses local files instead of querying trunk
- **Uses RN-tested versions** - glog 0.3.5 (not 0.3.4) with Xcode 26+ compatibility
- **Deterministic builds** - Same specs on all machines (local, CI, team members)

### Post-Install Hardening

The Podfile also includes modern build settings to exclude ancient architectures:

```ruby
post_install do |installer|
  react_native_post_install(
    installer,
    config[:reactNativePath],
    :mac_catalyst_enabled => false,
    :ccache_enabled => ccache_enabled?(podfile_properties),
  )

  # Modern Xcode hardening: exclude armv7/armv7s, set deployment target
  installer.pods_project.targets.each do |target|
    target.build_configurations.each do |config|
      # Exclude ancient architectures
      config.build_settings['EXCLUDED_ARCHS[sdk=iphoneos*]'] = 'armv7 armv7s'

      # Set minimum deployment target to avoid Xcode 26+ compatibility issues
      deployment_target = config.build_settings['IPHONEOS_DEPLOYMENT_TARGET']
      if deployment_target && deployment_target.to_f < 13.0
        config.build_settings['IPHONEOS_DEPLOYMENT_TARGET'] = '13.0'
      end
    end
  end
end
```

## ⚠️ CRITICAL: Expo Prebuild Caveat

**DO NOT run `expo prebuild --clean` without reapplying these modifications.**

The `ios/` directory is **gitignored** (generated by Expo) and `expo prebuild --clean` will regenerate a **vanilla Podfile without the overrides**.

### If You Need to Prebuild

```bash
# 1. Run prebuild
npx expo prebuild --platform ios --clean

# 2. IMMEDIATELY reapply the podspec overrides to ios/Podfile
# (Add the pod declarations shown above inside the target block)

# 3. Then install
cd ios
pod install --repo-update
cd ..
```

### CI Handles This Automatically

The `.github/workflows/ci.yml` workflow automatically injects these overrides after `expo prebuild`:

```yaml
- name: Fix Podfile for CocoaPods specs
  run: |
    cd ios
    # Add CDN source
    echo "source 'https://cdn.cocoapods.org/'" | cat - Podfile > Podfile.tmp
    mv Podfile.tmp Podfile

    # Inject local podspec overrides via awk
    awk '/use_expo_modules!/ {
      print;
      print "  pod '\''RCT-Folly'\'', :podspec => ...";
      # ...etc
    }' Podfile > Podfile.tmp
    mv Podfile.tmp Podfile
```

## Common Errors & Solutions

### "Unable to find a specification for fast_float (= 8.0.0)"

**Cause:** CocoaPods trying to resolve from trunk, spec not found or outdated

**Fix:** Add local podspec overrides (see above)

### "configure: error: C compiler cannot create executables" (glog)

**Cause:** glog 0.3.4 from trunk tries to build for armv7, fails on modern Xcode

**Fix:**

1. Add glog local podspec override
2. Add post_install hardening to exclude armv7
3. Clean and reinstall:
   ```bash
   cd ios
   rm -rf Pods Podfile.lock
   pod install --repo-update
   ```

### "missing: Unknown `--is-lightweight' option" (glog)

**Cause:** Ancient glog build scripts incompatible with modern automake

**Fix:** Same as above - use React Native's glog 0.3.5 from local podspec

## Verification

After `pod install`, check that local podspecs were used:

```bash
cd ios
grep "Fetching podspec for \`glog\`" Podfile.lock
# Should show: from `../node_modules/react-native/third-party-podspecs/glog.podspec`

grep "glog" Podfile.lock | head -1
# Should show: - glog (0.3.5)  <-- NOT 0.3.4
```

## Why Not Upgrade React Native?

React Native 0.82+ has breaking changes and Expo SDK 54 (our current version) is locked to RN 0.81.5. Upgrading would require:

- Expo SDK upgrade (coordinated dependency changes)
- Breaking changes in React Native APIs
- Potential native module incompatibilities

The local podspec approach is **simpler, safer, and zero-risk** compared to a major version upgrade.
